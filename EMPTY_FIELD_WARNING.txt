Date: January 29, 2026

FEATURE OVERVIEW
================================================================================

Added visual warning indicators to alert users when required date picker fields
are left empty. The warning appears dynamically when:

1. Field is empty (no date selected)
2. User hasn't filled the required field
3. Visual cues guide user to fill it

================================================================================
WARNING COMPONENTS
================================================================================

1. WARNING BADGE (Top Right):
   • Red background (Color(0xFFFFEBEE))
   • Red border (Colors.redAccent.withOpacity(0.3))
   • Warning icon + "Required" text
   • Only shows when field is empty AND it's a date picker
   • Disappears when user selects a date

2. BORDER COLOR:
   • Normal: Colors.grey.shade300
   • Warning: Colors.redAccent.withOpacity(0.5)
   • Thicker when warning (1.5 instead of 1)

3. ICON COLOR:
   • Normal: AppColors.primary (blue)
   • Warning: Colors.redAccent (red)

4. SHADOW EFFECT:
   • Normal: Black (0.05 opacity)
   • Warning: Red (0.1 opacity) - more visible

5. INPUT ICON (Calendar):
   • Normal: Colors.grey.shade400
   • Warning: Colors.redAccent

================================================================================
CODE IMPLEMENTATION
================================================================================

CHANGES MADE:

1. Added label header with warning badge:
   Row(
     mainAxisAlignment: MainAxisAlignment.spaceBetween,
     children: [
       Text(label, ...),
       ValueListenableBuilder<TextEditingValue>(
         builder: (context, value, child) {
           if (onTap != null && value.text.isEmpty) {
             return warningBadge(); // ← Shows warning
           }
           return SizedBox.shrink(); // ← Hidden when filled
         },
       ),
     ],
   )

2. Wrapped entire container with ValueListenableBuilder:
   ValueListenableBuilder<TextEditingValue>(
     valueListenable: controller,
     builder: (context, value, child) {
       final isEmpty = value.text.isEmpty;
       final hasWarning = onTap != null && isEmpty;
       
       return Container(
         decoration: BoxDecoration(
           border: Border.all(
             color: hasWarning 
               ? Colors.redAccent.withOpacity(0.5)
               : Colors.grey.shade300,
             width: hasWarning ? 1.5 : 1,
           ),
           boxShadow: [
             BoxShadow(
               color: hasWarning
                 ? Colors.redAccent.withOpacity(0.1)
                 : Colors.black.withOpacity(0.05),
             ),
           ],
         ),
         // ... field content
       );
     },
   )

3. Conditional icon color:
   Icon(
     icon,
     color: hasWarning ? Colors.redAccent : AppColors.primary,
     size: 18,
   )

4. Calendar icon changes color on warning:
   Icon(
     Iconsax.calendar,
     color: hasWarning 
       ? Colors.redAccent 
       : Colors.grey.shade400,
   )

REACTIVE BEHAVIOR
================================================================================

The warning uses ValueListenableBuilder, so:

✅ Warning shows IMMEDIATELY when field is empty
✅ Warning disappears INSTANTLY when date is selected
✅ No manual refresh needed
✅ No GetX dependency needed
✅ Pure Flutter reactive pattern
✅ Efficient - only affected widget rebuilds

================================================================================
FIELDS WITH WARNINGS
================================================================================

Date-based fields (onTap != null):
  • Preferred Disconnection Date ← Has warning
  
Regular text fields:
  • Reason for Disconnection ← No warning badge (open text field)
  • Bank Account Number ← No warning badge (open text field)
  • IFSC Code ← No warning badge (open text field)
  • Bank Registered Name ← No warning badge (open text field)
  • FTTH Number ← No warning badge (open text field)

Note: Only date picker fields show the warning badge because they require
explicit user action (tap to select). Regular text fields can be typed into.

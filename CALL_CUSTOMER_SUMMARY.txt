📱 FEATURE OVERVIEW
═══════════════════════════════════════════════════════════════════════════

When a technician clicks "Make Call" on a customer ticket:
  1. App sends call request to server with technician ID + customer mobile
  2. Server receives call request and initiates call to customer
  3. CallScreen opens on technician's phone
  4. Customer receives incoming call from server

═══════════════════════════════════════════════════════════════════════════
🔌 API ENDPOINT
═══════════════════════════════════════════════════════════════════════════

URL:      {{base_url}}/af/api/call_customer.php
Method:   POST
Parameters:
  - Technician_id: 133
  - mobile_no: "8360977765" (customer mobile number)

✨ NEW METHOD: callCustomer()
═══════════════════════════════════════════════════════════════════════════

Location: TechnicianAPI class
Returns:  Future<Map<String, dynamic>?>
Purpose:  Send call request to server with customer mobile number

Signature: Future<Map<String, dynamic>?> callCustomer({required String mobileNo})

What it does:
  ✅ Gets technician ID from SharedPreferences
  ✅ Sends POST request to call_customer.php with {Technician_id, mobile_no}
  ✅ Logs request/response details with customer mobile
  ✅ Returns parsed JSON response
  ✅ Handles errors gracefully

Example Request Body:
  {
    "Technician_id": 133,
    "mobile_no": "8360977765"
  }

Example Response:
  Success: {"status": "success", "message": "Call initiated..."}
  Error:   {"status": "error", "message": "Failed to initiate call"}
